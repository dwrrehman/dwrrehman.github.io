This is an Artificial Intelligence created by Daniel Rehman.

What is its purpose?

Its purpose is to interact with humans in the languages fubau and Javascript in a human-like way.

How do you use it?

There are four main input types you can say to it:

- Human language
- Computer language
- Commands
- Shortcuts

Human language:

This type of input should be said completely in the langauge of fubau, but can be extremely freeform. all spaces are removed, as well as numerals, special characters, (except for periods) and all letters are converted to lowercase. also, any stray consonants are removed from the input if they do not contribute to a word, which resolves most typos. a dictionary and tutorial is available if you need help speaking the language intelligeablely.

Computer language:

This artifical intelligence has the unique capability of speaking two very different languages. the AI also knows the computer programming language Javascript, so any ad-hoc coding you wish to do can be executed in the input stream, simply by entering Javascript mode. to enter JS mode, you can either press "j" and then enter, or type the command "//" and then enter. these both put you into JS mode for you to type any amount of vaild javascript into the textarea. to execute the code and return to fubau mode, you simply must type "//", just as you may have went into JS mode. because of this architecure, JS single-line comments are prohibited. other than this, every Javascript functionality should be built into the AI.

Commands:

Built into this artificial intelligence are a list of commands that execute various functionalities that might be of use. here is the list of possible commands:

- /    :   an empty command, or some special functionality.
- //    :   enter Javascipt mode.(must be root)
- /.   :   an initiation command.
- /login   :   login to the user database.
- /clear   :   clear the screen, but keep the conversation in memory.
- /add   :   add an entry to the dictionary, or some other domain.(must be root)
- /kill   :   kill the artifical intelligence by closing the window.(must be root)
- /display conversation   :    display the current conversation.
- /display dictionary   :    display every word in the dictionary.
- /display predicates   :    display all the legal predicates.
- /display previous commands   :    display the commands you've typed.
- /display max word length   :   display the lexer's maximum length of a word it will accept. (default is 3).
- /new   :   open this artifical intelligence again in a new window.
- /test   :   a test statement used for debugging.
- /who am i   :   display the user who's logged in at the moment.
- /reset   :    reset the artifical intelligence by reloading the web page.
- /info   :    open this info page.
- /list   :    display a list of these commands.
- /set max word length   :    change the lexer's maximum length of a word it will accept. (must be root)
- /break   :    effectively break the artifical intelligence by deleteing all the html. (must be root)
- /hide   :   hide the user interface until a key is pressed.

Shortcuts:

These are a small list of single letter commands which are generally used so much they were warrented a shortcut to them, as opposed to typing out the whole command every time. the shortcuts are as follows:

- u   :    signify the end of transmission, and close the AI.
- x   :    kill the AI by force closing it.
- h   :    hide the user interface until a key is pressed.
- j   :    enter Javascript mode.
- r   :    reset the AI by reloading the web page.
- c   :    clear the screen but keep the conversation.
- d   :    display the current conversation so far.
- l   :    list the possible commands you can tell it.

What is it comprised of?

The artificial intelligence and everything it entails is composed of 3 main sections with subsections:

- Checker
   - Executer
- Parser
   - Lexer
   - Labeler
   - Other grammar functions
- Responder
   - Semantic database
   - Neural network

Checker:

this function gets the users input before any other function, and is a series of if-statements which check to see if the input is one of a few special commands. these commands are mainly command-line interface functions, like clearing the screen, printing the current conversation, and closing the window. there is also a spot to where a user defined functionality can lie.


Checker:Executer:

this function is a series of if-statements that some time implement a similar functionality as the checker, but other times has unique capabilities which make it important. it works by checking if the input has a forward slash at the beginning of the string. if it does, this means it is a non-linguistic command to be executed. after it knows it is a command, it runs through some if statements and checks each possible command.


Parser:

this is functions job is primarily to understand the language of choice for this artificial intelligence, which is fubau.fubau is made to be completely unambiguous, as to make the parser much simpler. the two major things the parser has to accomplish are lexing the input, so identifying all strings in the input that are also contained within the dictionary, and labeling the input, which consists of identifying all grammtical constructs and labeling it with the name of that respective construct in the form of an object. this sets the input up for the responder to interpret it and make a response.
